## 1. Visual Theme & Atmosphere

- **ミッション**: 「誰一人取り残されない、人に優しいデジタル化を。」
- **対象**: 官公庁・地方自治体・公共性の高い組織のWebサイト／Webアプリ／オンラインサービス
- **トーン**: 誠実・明快・中立・信頼感。装飾より機能性を優先する。
- **デザイン密度**: 余裕のある余白を確保し、情報過密を避ける。
- **アクセシビリティ最優先**: WCAG 2.2 AA準拠（一部AAA）、カラーユニバーサルデザイン配色を遵守。
- **レスポンシブ**: モバイルファーストで設計する。

## 2. Color Palette & Roles

### カラー体系の考え方

デザインシステムのカラーは以下の4層で構成される。  
個別サービスはこの体系に基づき、プライマリーカラーの色相を選んでスキームを構築する。

```
プリミティブカラー（パレット原色）
  └── キーカラー（Primary / Secondary / Tertiary / Background）
        └── 共通カラー（ニュートラル：白〜グレー〜黒）
              └── 機能カラー / アクセントカラー / セマンティックカラー
```

### プリミティブカラー（参照パレット）

各色相は13段階の階調を持つ。以下は代表値（100=最も明るい、1300=最も暗い）。

| 色相         | 代表プライマリー値（例） |
|--------------|--------------------------|
| Blue（青）   | #0017C1（Blue-900相当）  |
| Light Blue   | #0080CC                  |
| Cyan         | #008080                  |
| Green（緑）  | #1B7A3E                  |
| Orange       | #C84B00                  |
| Red（赤）    | #C01A00                  |
| Purple（紫） | #6600CC                  |

> 具体的なHEX値はFigmaデータ（design-tokens）を正とする。  
> npm: `@digital-go-jp/design-tokens`、CSS Custom Propertiesとして提供。

### キーカラー

| ロール            | 説明                                                      | コントラスト要件       |
|-------------------|-----------------------------------------------------------|------------------------|
| Primary           | ブランドの主色。ヘッダー・CTAボタン・アクティブ状態に使用 | 背景に対して **4.5:1以上** |
| Secondary         | Primaryを補完。副次的なボタンや状態表示                   | 4.5:1以上（テキスト時）、3:1以上（非テキスト時） |
| Tertiary          | SecondaryとPrimaryの間の明度差で設定                      | 同上                   |
| Background        | セクション背景として白・黒以外を使う場合                  | 上に載るテキストとのコントラストを再確認 |

### 共通カラー（ニュートラルカラー）

白〜黒の14階調グレースケール。テキスト・境界線・背景・UI部品に使用。

| トークン名             | 用途                    | 目安値      |
|------------------------|-------------------------|-------------|
| Neutral-0 / White      | ページ背景（標準）       | `#FFFFFF`   |
| Neutral-200            | 薄いサーフェス背景       | `#F2F2F2`   |
| Neutral-400            | 境界線・ディバイダー     | `#C1C1C6`   |
| Neutral-600            | 補助テキスト・disabled  | `#767680`   |
| Neutral-800            | 本文テキスト（標準）     | `#3B3B3F`   |
| Neutral-1000 / Black   | 見出し・強調テキスト     | `#1A1A1C`   |

> テキスト ↔ 背景は常に **4.5:1以上**  
> 境界線等の非テキスト要素は **3:1以上**

### 機能カラー（Functional Colors）

| ロール        | 用途                              | 色相の目安  |
|---------------|-----------------------------------|-------------|
| Link          | リンクテキスト                    | Blue系      |
| Visited Link  | 訪問済みリンク                    | Purple系    |
| Focus         | キーボードフォーカスリング        | Blue / Cyan |
| Hover         | ホバー状態                        | Primary の明度調整 |
| Disabled      | 操作不可状態                      | Neutral-600 |

### セマンティックカラー

| ロール        | 意味             | 色相の目安  |
|---------------|------------------|-------------|
| Error / Danger | エラー、危険     | Red         |
| Warning        | 警告、注意       | Orange / Yellow |
| Success        | 完了、正常       | Green       |
| Info           | 情報、お知らせ   | Blue        |

## 3. Typography

### フォントファミリー

```css
body {
  font-family: 'Noto Sans JP', -apple-system, BlinkMacSystemFont, sans-serif;
}

code, pre {
  font-family: 'Noto Sans Mono', monospace;
}
```

- **Noto Sans JP**: 基本書体。可読性・視認性の高いサンセリフ。SIL Open Font License 1.1 / Webフォントとして利用可能。
- **Noto Sans Mono**: コード・等幅表示用。

### フォントウェイト

| レベル | `font-weight` | 用途              |
|--------|--------------|-------------------|
| N（Normal） | `400`   | 本文・通常テキスト |
| B（Bold）   | `700`   | 見出し・強調      |

### フォントサイズスケール

| トークン | px値  | 用途の目安                          |
|----------|-------|-------------------------------------|
| xs       | 11px  | 補足・キャプション（最小）          |
| sm       | 12px  | ラベル・チップ等                    |
| md-s     | 14px  | 補助テキスト・フォームラベル        |
| md       | 16px  | 本文（標準）                        |
| lg-s     | 18px  | 小見出し                            |
| lg       | 20px  | 中見出し                            |
| xl       | 24px  | 大見出し（h3相当）                  |
| 2xl      | 28px  | セクション見出し（h2相当）          |
| 3xl      | 32px  | ページ見出し（h1相当）              |
| 4xl      | 36px  | ヒーロー見出し                      |
| 5xl      | 40px  | 特大見出し                          |
| 6xl      | 48px  | ディスプレイ                        |

### 行間隔（`line-height`）

| 用途           | 推奨値        |
|----------------|--------------|
| 本文テキスト   | `1.7` 〜 `1.8`（日本語の可読性確保） |
| 見出し         | `1.3` 〜 `1.5` |
| コード         | `1.6`        |

### アクセシビリティ（テキスト）

- フォントサイズは最小 **14px** 以上を本文に使用する（WCAGガイドライン）
- ユーザーがブラウザでフォントサイズを200%に拡大しても、コンテンツ・機能が損なわれないようにする
- テキストのみのズーム操作をサポートする

## 8. Components

### ボタン（Button）

| バリアント    | 用途                    | スタイル方針                      |
|---------------|-------------------------|-----------------------------------|
| Primary       | 主要アクション（CTA）   | プライマリーカラー背景・白テキスト |
| Secondary     | 副次アクション          | 境界線あり・プライマリーカラーテキスト |
| Tertiary      | 補助アクション          | テキストのみ                      |
| Danger        | 削除・取り消し等        | Errorカラー系                     |

- **高さ**: Small=32px、Medium=40px（標準）、Large=48px
- **タッチターゲット**: 最小 **44×44px**（モバイルアクセシビリティ）
- **フォーカスリング**: 必須。`outline` で視認性を確保する。
- **disabled状態**: Neutral-600相当のテキスト・背景、`cursor: not-allowed`

### フォームコンポーネント

- **インプットテキスト / テキストエリア**: 境界線は通常 `2px solid`（Neutral-400）、フォーカス時にPrimaryカラーの境界線
- **エラー状態**: Errorカラーの境界線＋エラーメッセージを入力欄の直下に表示
- **ラベル**: 入力欄の上に配置。`required` には「必須」バッジを付与する
- **セレクトボックス / ラジオボタン / チェックボックス**: ネイティブ要素を拡張してスタイリング。WAI-ARIAに準拠

### ナビゲーション

- **グローバルメニュー**: ヘッダー上部に配置。PC版はホリゾンタル、モバイルはドロワーやハンバーガーメニュー
- **パンくずリスト**: ページ上部に配置、区切り文字は `>`
- **ステップナビゲーション**: 手続きフローを明示。現在ステップをハイライト
- **ページナビゲーション（ページネーション）**: 前後ボタン＋ページ番号。ボタン高さはMedium（40〜48px）

### カード（Card）

- `border-radius: radius-lg`（8px）推奨
- 画像・テキスト・CTAリンクで構成
- リンク全体がクリック可能なカードは `<a>` で囲む（インタラクティブカード）
- hover時は軽いelevationまたはborder色の変化

### テーブル

- ヘッダー行は `<thead>` + `<th scope="col">` を必ず使用
- 縞模様（zebra stripe）で可読性向上
- モバイルでは横スクロールを許容する

### モーダルダイアログ

- フォーカストラップを実装（Tabキーがダイアログ内に留まる）
- 開閉時にフォーカスを適切に移動
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby`

### ノティフィケーションバナー（通知）

| タイプ   | セマンティックカラー | `role`           |
|----------|----------------------|------------------|
| Error    | Error / Red          | `role="alert"`   |
| Warning  | Warning / Orange     | `role="alert"`   |
| Success  | Success / Green      | `role="status"`  |
| Info     | Info / Blue          | `role="status"`  |

## 9. Icons

- **ライブラリ**: Material Symbols（Googleフォントアイコン）を推奨
- **サイズ**: 20px（sm）、24px（md、標準）、32px（lg）
- **代替テキスト**: 意味を持つアイコンには `aria-label` または隣接テキストを付与
- **装飾アイコン**: `aria-hidden="true"` を付与してスクリーンリーダーに無視させる

## 10. Accessibility Requirements

- **WCAG準拠レベル**: 2.2 AA（一部 AAA）
- **コントラスト比**:
  - テキスト: **4.5:1以上**（小テキスト）、**3:1以上**（大テキスト 18px以上・bold 14px以上）
  - 非テキストUI（アイコン・境界線・フォームコントロール）: **3:1以上**
- **キーボード操作**: 全インタラクティブ要素はキーボードのみで操作可能にする
- **フォーカス可視化**: `outline` を `none` にしない。フォーカスリングを明示する
- **スクリーンリーダー対応**: 適切な `role`・`aria-*` 属性・`alt` テキストを付与
- **色だけで情報を伝えない**: アイコン・テキスト・パターン等を併用する
- **動き・アニメーション**: `prefers-reduced-motion` を尊重し、過度なアニメーションを抑制

## 11. Do's & Don'ts

### Do（推奨）

- デザイントークンを使ってカラー・スペーシング・タイポグラフィを一元管理する
- セマンティックHTMLを優先する（`<button>`・`<nav>`・`<main>` 等）
- フォームのエラーメッセージは入力欄の直下に、具体的な内容で表示する
- ラベルとフォームコントロールを `<label for>` または `aria-labelledby` で明示的に紐付ける
- コンポーネントの最小タッチターゲットサイズ（44×44px）を守る
- テキスト200%拡大でもレイアウトが破綻しないようにする

### Don't（非推奨）

- テキストに `outline: none` / `outline: 0` を設定してフォーカスリングを消す
- 色だけでエラーや状態を伝える（例：赤い枠線のみ）
- 意味のある画像の `alt` を空にする
- ボタンの中に `<a>`・`<div>` をネストする（またはその逆）
- コントラスト比を満たさない薄いグレーテキストをプレースホルダーや補助テキストに使う
- ユーザーが操作していないのに自動的にコンテンツをスクロール・移動させる
